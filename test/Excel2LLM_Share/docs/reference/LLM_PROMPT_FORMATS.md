# Excel2LLM 向け LLM 指示フォーマット集

- 作成者: Codex (GPT-5)
- 更新日: 2026-03-28

## この文書の目的

この文書は、`Excel2LLM` で作成した `llm_package.jsonl` や `workbook.json` を使って、LLM に何をどう依頼すればよいかを標準化するためのテンプレート集です。

単なる例文ではなく、毎回コピーして使える「記入式フォーマット」としてまとめています。

## テンプレート選択の早見表

最初に「何をしたいか」で選ぶと迷いにくいです。

| やりたいこと | 使うテンプレート |
| --- | --- |
| 2 つの Excel や 2 つの表を付き合わせて結合したい | `1. 2 つの Excel の結合` |
| 表から傾向、異常値、欠損、統計的な要点を見たい | `2. 表からのデータ抽出、統計分析` |
| 機械設計の計算手順、数式の妥当性、最適化余地を見たい | `3. 機械設計の計算シート分析、計算手順確認、計算最適化` |
| 会計管理表の差異、利益率、確認ポイントを見たい | `4. 会計管理表の分析、内容確認` |
| シート全体を見て改善案、数式提案、処理手順改善を出したい | `5. 総括的な Excel 改善` |

迷ったときは、まず `5. 総括的な Excel 改善` を使い、その後で必要に応じて専門テンプレートへ分けてください。

## まず結論

LLM への指示は、次の 6 点を毎回はっきり書くと安定します。

1. 何をしたいか
2. どのシートや範囲を見るか
3. `formula` / `formula2` / `value2` / `text` のどれを優先して見るか
4. どこまで推測してよいか
5. どんな形式で答えてほしいか
6. 不足情報がある場合にどう振る舞ってほしいか

## 基本フォーマット

まずはこの基本形を使ってください。

```text
あなたは Excel 分析支援アシスタントです。

目的:
- [ここに依頼内容を書く]

対象:
- 対象ファイル: [ファイル名または案件名]
- 対象シート: [sheet_name または "全シート"]
- 対象範囲: [range または "対象チャンク全体"]

データの見方:
- 数式は formula と formula2 を優先して確認する
- 値は value2 を基準に確認する
- 表示上の見え方が必要なときだけ text を参照する
- style 情報は補助扱いとし、判断の主根拠にはしない

作業指示:
- [してほしい分析や処理を箇条書きで書く]

出力形式:
- [欲しい出力形式を書く]

制約:
- 根拠のない推測はしない
- 不明点は不明と書く
- 数式に基づく説明は、対象セルや列の関係が分かるように書く
- 必要なら改善案を出すが、元データを破壊する前提では提案しない

入力データ:
[ここに llm_package.jsonl の 1 チャンク、または workbook.json の一部を貼る]
```

## 出力形式の指定テンプレート

迷ったら、次のいずれかを使ってください。

### 短い要約がほしい場合

```text
出力形式:
- まず 3 行以内で全体要約
- 次に重要ポイントを 5 点以内で列挙
- 最後に注意点を 3 点以内で列挙
```

### 詳細レビューがほしい場合

```text
出力形式:
- 1. 概要
- 2. 発見事項
- 3. 数式や処理の読み解き
- 4. 問題点
- 5. 改善案
- 6. 確認が必要な点
```

### 表形式で整理してほしい場合

```text
出力形式:
- 次の列を持つ表で出力する
- 対象セル/対象範囲
- 現状
- 問題または気づき
- 推奨対応
- 根拠
```

## 共通で付けるとよい補助指示

### 数式重視

```text
数式セルは formula または formula2 を優先して確認し、text だけで判断しないでください。
```

### 表示値重視

```text
利用者が画面で見ている値を重視したいので、value2 だけでなく text の違いも確認してください。
```

### 推測を抑えたい

```text
列名や用途が明示されていない場合は、推測で断定せず、候補として表現してください。
```

### 改善案までほしい

```text
問題点の指摘だけでなく、すぐ実行できる改善案を優先順位付きで提案してください。
```

## 用途別テンプレート

以下は、今回のご要望に合わせた実務テンプレートです。

### 1. 2 つの Excel の結合

#### 使いどころ

- 顧客一覧と売上一覧を付き合わせたい
- マスタと実績を 1 つにまとめたい
- シート名や列名が少し違う 2 つの Excel を統合したい

#### テンプレート

```text
あなたは Excel 統合支援アシスタントです。

目的:
- 2 つの Excel データを結合する方針を整理し、結合キー、前処理、結合後の注意点を提案してください。

対象:
- ファイルA: [ファイルA名]
- ファイルB: [ファイルB名]
- 対象シート: [対象シート名]

データの見方:
- 列名、sheet_name、value2、text を確認する
- 数式列がある場合は formula / formula2 も確認する
- 同じ意味だが表記ゆれしている列候補も洗い出す

作業指示:
- 結合候補となるキー列を提案する
- 列名の表記ゆれ、型の違い、空欄、重複の有無を確認する
- 内部結合、左結合、全結合のどれが適切か提案する
- 結合後に欠損や重複が起きそうな箇所を指摘する
- 必要なら結合前の整形ルールを提案する

出力形式:
- 1. 結合の目的
- 2. 推奨キー列
- 3. 事前整形の提案
- 4. 推奨結合方法
- 5. リスク
- 6. 実施手順

制約:
- キーが不十分なら、その旨を明記する
- 推測だけで「この列が絶対に主キー」と断定しない

入力データ:
[ファイルAのチャンク]
[ファイルBのチャンク]
```

#### 期待する答え

- 何をキーに結合すべきか
- どの列を事前に正規化すべきか
- 結合後に壊れやすい点は何か

### 2. 表からのデータ抽出、統計分析

#### 使いどころ

- 売上表から月次集計をしたい
- 在庫表から異常値を探したい
- アンケート結果の傾向を見たい

#### テンプレート

```text
あなたは Excel データ分析アシスタントです。

目的:
- 表データから重要な情報を抽出し、統計的な観点で要点を整理してください。

対象:
- ファイル名: [ファイル名]
- 対象シート: [sheet_name]
- 対象範囲: [range]

データの見方:
- value2 を基準に数値分析する
- text にしか現れない表示上の差異があれば補足する
- 列の意味はヘッダー候補と周辺セルから推定する

作業指示:
- 列の意味を整理する
- 集計に使える列とカテゴリ列を分ける
- 基本統計の観点で見て重要な傾向をまとめる
- 外れ値、欠損、重複、形式ゆれを指摘する
- 追加で見るべき切り口を提案する

出力形式:
- 1. 表の概要
- 2. 主な列の意味
- 3. 主要な傾向
- 4. 異常値や欠損の候補
- 5. 追加分析の提案

制約:
- 実際に計算できる範囲だけで結論を書く
- データ不足なら不足点を明記する

入力データ:
[対象チャンク]
```

### 3. 機械設計の計算シート分析、計算手順確認、計算最適化

#### 使いどころ

- 強度計算、寸法計算、モータ選定計算などのレビュー
- 計算順序が妥当か確認したい
- 冗長な数式や分かりにくい中間計算を改善したい

#### テンプレート

```text
あなたは機械設計向けの計算シートレビューアシスタントです。

目的:
- 計算シートの構造、計算手順、数式の役割を読み解き、妥当性と改善余地を整理してください。

対象:
- ファイル名: [ファイル名]
- 対象シート: [sheet_name]
- 対象範囲: [range または "関連チャンク全体"]

データの見方:
- formula / formula2 を最優先で確認する
- value2 と text の差があれば単位や表示丸めの可能性も考慮する
- 入力値セル、中間計算セル、最終結果セルを区別して整理する

作業指示:
- 計算の流れを上流から下流まで説明する
- 入力値、中間値、最終出力を分類する
- 数式の依存関係が不自然な箇所を指摘する
- 重複計算、冗長計算、分かりにくい参照を洗い出す
- 計算シートとしての可読性や保守性の改善案を提案する
- 必要なら計算式の整理方針や補助列の追加案を出す

出力形式:
- 1. シートの目的
- 2. 計算手順の流れ
- 3. 主要セルの役割
- 4. 問題点
- 5. 最適化案
- 6. 確認が必要な設計前提

制約:
- 機械設計上の前提条件が不足している場合は断定しない
- 安全率、単位、境界条件は推測で埋めない

入力データ:
[対象チャンク]
```

### 4. 会計管理表の分析、内容確認

#### 使いどころ

- 部門別損益の確認
- 月次推移の読み解き
- 数式監査前のチェック

#### テンプレート

```text
あなたは会計管理表レビューアシスタントです。

目的:
- 会計管理表の構造、集計ロジック、異常値候補、確認すべき箇所を整理してください。

対象:
- ファイル名: [ファイル名]
- 対象シート: [sheet_name または "全シート"]

データの見方:
- 集計セルは formula / formula2 を優先して確認する
- 金額分析は value2 を基準に行う
- 表示丸めや書式差異の確認に text を補助利用する

作業指示:
- シートごとの役割を整理する
- 集計ロジックの流れを説明する
- 前月比、部門差、異常値候補を指摘する
- 入力ミスの可能性がある箇所を挙げる
- 追加確認すべき点を提案する

出力形式:
- 1. 全体概要
- 2. シート別の役割
- 3. 数値上の気づき
- 4. 数式・集計上の注意点
- 5. 追加確認項目

制約:
- 監査結論のような断定はしない
- 元帳や証憑がない限り、事実確定ではなく確認候補として出す

入力データ:
[対象チャンク]
```

### 5. 総括的な Excel 改善

#### 使いどころ

- 数式提案
- 処理手順の提案
- シート構成の見直し
- 全体の見た目や使いやすさの改善

#### テンプレート

```text
あなたは Excel 改善提案アシスタントです。

目的:
- 現在の Excel の構造、数式、運用方法、見た目を踏まえて、総合的な改善案を提案してください。

対象:
- ファイル名: [ファイル名]
- 対象シート: [sheet_name または "全体"]

データの見方:
- シート構成、列構成、formula / formula2、value2、text を確認する
- styles 情報がある場合は見た目の補助根拠として使う

作業指示:
- 現状の問題点を分類する
- 数式改善案を提案する
- 処理手順や入力フローの改善案を提案する
- シート分割や統合の見直し案を出す
- 見た目の改善案を提案する
- 優先順位を付ける

出力形式:
- 1. 現状の総評
- 2. 問題点の分類
- 3. 数式改善案
- 4. 運用改善案
- 5. 見た目改善案
- 6. 優先順位付き実施計画

制約:
- 実装コストが高い提案は、その旨を明記する
- 見た目改善だけでなく、保守性や誤入力防止も考慮する

入力データ:
[対象チャンク]
```

## 運用上のおすすめ

- 重要なブックは、先に `Excel2LLM.bat -Verify` を通してから LLM に渡す
- LLM には `workbook.json` 全量ではなく、`llm_package.jsonl` の必要チャンクだけを渡す
- style は補助扱いにし、値や数式の判断を優先する
- 1 回で全部を聞かず、「構造理解」「問題抽出」「改善案」を分けて聞くと精度が上がりやすい
